# Architecture

Architecture notes live here.

Use this directory for cross-cutting design documents that are broader than a
single library or executable.

Use `docs/adr/` for durable architectural decisions. Use this directory for
living architecture notes that explain how systems fit together after those
decisions have been made.

Library-specific ownership and dependency boundaries live in each
`libs/<library>/docs/Boundary.md` file. Executable-specific guidance lives in the
matching executable `docs/` directory.

Generated API documentation is deferred.
