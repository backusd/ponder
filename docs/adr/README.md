# Architecture Decision Records

Lightweight ADRs live here.

Use ADRs for durable decisions that should be remembered after the immediate
implementation context is gone.

## Format

Use short Markdown files named:

```text
NNNN-kebab-case-title.md
```

Each ADR should include:

- Status.
- Context.
- Decision.
- Consequences.
