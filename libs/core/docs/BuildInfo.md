# Core Build Information

`BuildInfo` exposes a small snapshot of build metadata through
`#include <ponder/core/BuildInfo.hpp>`.

## Generation

The implementation source is generated by CMake during configure using
`ponder_configure_build_info` and `configure_file`.

Generated files are written under the active build tree, normally:

```text
build/<preset>/libs/core/generated/src/BuildInfo.cpp
```

The generated source is not checked in. If checked-in generated files are ever
introduced, the generator version and regeneration command must be documented
next to those files.

## Values

- Project name and project version come from the root `project()` declaration.
- Git commit comes from `git rev-parse --verify HEAD` when Git and repository
  metadata are available during CMake configure. It is empty otherwise.
- Build type is supplied from the active CMake build configuration when the
  generated source is compiled.
- Compiler name/version come from CMake's detected C++ compiler metadata.
- Target system, target processor, and pointer width come from CMake platform
  detection.
- CMake generator and CMake version describe the configure step that produced
  the generated source.

Build information is configure-time metadata. Re-run CMake configure when the
Git commit or compiler/toolchain metadata must be refreshed.
