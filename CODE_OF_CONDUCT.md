# Code Of Conduct

`ponder` should be a serious, welcoming, and durable open-source project. Treat
other contributors with respect, assume good intent where reasonable, and keep
technical disagreement focused on the work.

## Expected Behavior

- Be respectful and constructive.
- Prefer clear explanations over dismissive comments.
- Make room for contributors with different backgrounds and experience levels.
- Keep discussions focused on improving the project.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or conduct.
- Publishing private information without permission.
- Repeated disruption of technical discussion.

## Enforcement

Before the project is public, maintainers will handle conduct concerns through
the private development channel. Before public launch, this file should be
updated with the project's public reporting and enforcement process.
